# GZTips

Repositório de dicas, tutoriais e anotações rápidas para o dia a dia de desenvolvimento e operação. A ideia é centralizar conteúdos curtos e úteis em um só lugar, de forma organizada e fácil de consultar.

## Objetivo

- Agregar diferentes dicas/tutoriais úteis em um repositório único.
- Servir como referência rápida (copy/paste) para tarefas recorrentes.
- Evoluir de forma incremental conforme necessidades reais.

## Como usar

- Navegue pelos arquivos na raiz (cada tópico é um `.md` independente).
- Utilize a busca do GitHub/VSCode para encontrar termos específicos.
- Copie os trechos necessários e adapte ao seu contexto.

## Tópicos atuais

- VPS e DevOps básico:
  - Manutenção de VPS, comandos úteis e boas práticas
  - Arquivo: [VPS.md](./VPS.md)
- Git e Branching:
  - Fluxos de trabalho, convenções, criação/gerenciamento de branches
  - Arquivo: [GITHUB - BRANCH.MD](./GITHUB%20-%20BRANCH.MD)

## Estrutura sugerida (guides)

Cada tópico deve preferir a estrutura abaixo (quando fizer sentido):

1. Contexto/Objetivo (o que resolver)
2. Pré-requisitos (versões, permissões, ferramentas)
3. Passo a passo (comandos e explicações curtas)
4. Armadilhas comuns (gotchas) e troubleshooting
5. Referências (links externos úteis)

## Contribuindo

- Crie um novo arquivo `.md` para cada tema novo (nome claro e direto).
- Prefira conteúdo prático e exemplos mínimos funcionais.
- Mantenha a linguagem simples e objetiva.
- Use blocos de código com a linguagem correta para syntax highlight.
- Evite conteúdo duplicado; se existir, consolide e referencie.

Sugestões de novos tópicos bem-vindas:

- Shell scripting e aliases úteis
- Docker/Docker Compose (comandos do dia a dia)
- Node/PNPM/NPM scripts recorrentes
- CI/CD e GitHub Actions (snippets prontos)
- Observabilidade (logs, métricas, tracing) introdutório
- Banco de dados (backup/restore rápido, migrações)

## Convenções

- Idioma: Português (pt-BR).
- Formatação: Markdown simples, linhas curtas e listas objetivas.
- Comandos: preferir exemplos que funcionem em ambientes comuns (Ubuntu/Debian/Windows WSL quando aplicável).
- Segurança: cuidado ao publicar segredos; se necessário, exemplifique com placeholders.
